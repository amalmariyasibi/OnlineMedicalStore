## рҹҺҜ Key Points

### For Customer Upload to Work:
1. вң… User must be logged in
2. вң… File must be valid (image/PDF, <5MB)
3. вң… Cloudinary must be configured
4. вң… Firestore security rules must allow create
5. вң… User's userId must match document userId

### For Admin View to Work:
1. вң… Admin must be logged in
2. вң… Admin user must have `role: "admin"` in Firestore
3. вң… Firestore security rules must allow admin to read all
4. вң… Prescriptions collection must exist
5. вң… Prescriptions must have proper structure

## рҹ”Қ Common Issues

| Issue | Symptom | Solution |
|-------|---------|----------|
| Admin role not set | "User role is not admin" | Set `role: "admin"` in Firestore users collection |
| Permission denied | "рҹ”’ PERMISSION DENIED" | Update Firestore security rules |
| No prescriptions | "0 prescriptions" | Upload test prescription as customer |
| 404 on file | Image doesn't load | Check Cloudinary URL and configuration |
| Can't upload | Upload fails | Check Cloudinary API keys in backend |
